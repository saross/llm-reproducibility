> (1) Your project's alignment with the specific grant round (additional details here),

## Drafting implications

The written artefacts to produce, mapped to the framing doc
(`cosmos-grant-application-framing.md`):

1. **Self-pitch (1-2 sentences, field 6)** — archaeologist / open-science builder framing;
   draws on framing §4 differentiators (completed pilot; human-in-the-loop ergonomics).
2. **Project title (field 15)** — new; not yet drafted anywhere.
3. **One-sentence project description (field 16)** — compress framing §4 headline.
4. **Short proposal, under 500 words (field 17)** — must follow the five-part structure
   above. Map: (1) alignment ← framing §4 frame + tone; (2) problem ← reproducibility /
   verification gap, pilot headline finding; (3) medium ← code (pipeline + public
   dashboard / human-verification surface); (4) execution plan ← agentic modernisation plan
   phases, 90-day window (framing §7); (5) success case ← JAS census + reproduction verdicts
   public and checkable, preregistered confirmatory study.
5. **Grant request integer (field 14)** — refine from framing §6 brainstorm to one number.
6. **How-did-you-hear answer (field 20)** — factual; Shawn to supply.
7. **Optional-but-recommended fields** — the form explicitly says optional fields aid
   judging: CV upload, GitHub repository (decide which URL — check repo visibility first),
   website, LinkedIn, X handle, Substack. Collaborators (field 18) if any; project
   website / additional info (field 19) is a natural home for pilot-findings and
   variability-study pointers.

Everything except fields 1-5, 20, and the optional profile links is drafting work against
existing framing decisions — no new strategic decisions required. The half-day-artefact
estimate in framing §7 stands.
